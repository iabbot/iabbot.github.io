Edgemap

Highlights the edges on an image

Possible IDs:

  • Edgemap
  • Edge
  • 0

Parameters:

  • -hue

    Use the image's original colors

  • -radius #

    (Default 1) The number of pixels to search to consider a pixel an 'edge'. Typically makes lines thicker.

  • -tolerance #

    (Default 30) Defines how much a pixel's channels must change for it to be considered an edge.

  • -alphamode #

    (Default 1) Defines how to treat the alpha channel of the image.

    1. If the pixel is not an edge, its alpha is 0. Otherwise, it's 100.
    2. The pixel's alpha value is the same as the old image's.
    3. The pixel's alpha value is recalculated to be the difference between pixels. Often, you'll need to use -multiplier # to make it visible.

  • -multiplier #

    (Default 1) Multiplies the alpha channel by this amount. This number can have a decimal.

  • -bias #

    (Default 0) Adds or subtracts (if the value is negative) this number from the alpha channel. This is processed AFTER multiplication, and if the alpha value is 0, nothing will be added.

  • -color #

    (Default FFFFFF) A six-digit hex number (without the hex specifier) that defines the color of the edgemap (if -hue isn't specified), in RGB format. For example, FF0000 is red.



Fry

Distorts an image's colors by multiplying channel values' offsets from 128 by a given number.

Possible IDs

  • Fry
  • 1

Parameters

  • -i #

    (Default 5) The factor to multiply the channels by. Higher numbers do not necessarily mean more distortion (diminishing returns)

Saturate

Saturates an image

Possible IDs:

  • Saturate
  • 15

Parameters:

  • -saturation

    A value, ranging from 0 to 1, that describes the saturation. 0 is completely desaturated, 1 is completely saturated.

  • -mode

    A value that determines the mode to use:

    1. New pixel saturation = pixelSaturation * saturation
    2. New pixel saturation = saturation
    3. New pixel saturation = pixelSaturation + saturation

Chromakey

Cuts out a certain color from an image

Possible IDs:

  • Chromakey
  • CK
  • 17

Parameters:

  • -color

    A hex color, in RRGGBB format, that describes the color to remove

  • -tolerance

    The amount each pixel can differ from the key color and still be selected.

  • -alphamode

    The mode used to set the alpha:

    1. New pixel alpha = Alpha - alphaReduction
    2. New pixel alpha = old pixel alpha - (255 / (differenceR + differenceG + differenceB + 1))
      (typically gives worse results than alphamode 0)

  • -alphareduction

    Defines how much to reduce the selected pixels' alpha by. To completely cut out keyed pixels, use 255.
