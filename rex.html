iab!rex

This command manages runtime extensions.


A quick primer about runtime extensions (rEx)

Runtime extensions are the end-user's ability to add new commands (extensions) to iab while it's running (runtime).
There are a couple things to note:

  • You can only create up to four runtime extensions unless you become a plus user (see iab!plus)
  • Runtime extensions can only contain text (which can include an image link, but you cannot use it to upload a file.)
  • Runtime extensions cannot be created with the same name as an internal command (which may be changed at some point.)
  • You must be in the same server as the user who created a runtime command to use it.
  • You cannot create a runtime command with the same name as any other command. It is planned to develop a new command syntax for running runtime extensions that uses a username so that duplicate names can be allowed.

Runtime extensions can be run with the prefix rex! (rexd if using iabd)

Also I'm disclaiming responsibility for runtime extensions. If someone does something mean with it, it's not my fault.

Subcommands

Create

iab!rex create "trigger "response

"trigger represents the string that must be written after rex! to trigger the command. It cannot contain spaces.

"response is the response that iab should reply with.

Delete

iab!rex delete "trigger

"trigger represents the trigger of the command to delete..

Deleteall

iab!rex deleteall

Deletes all of your runtime extensions.

List

iab!info list [all]

Lists all of your runtime extensions, or all runtime extensions (in your guild) if 'all' is written.
